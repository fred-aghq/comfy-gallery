# ComfyUI Gallery

A self-hosted media gallery for browsing, viewing, and searching images and videos generated by [ComfyUI](https://github.com/comfyanonymous/ComfyUI).

## Features

- **Gallery Wall** — Responsive grid of thumbnails with lazy loading and pagination
- **Folder Navigation** — Browse your output directory tree structure
- **Media Viewer** — Full-resolution image viewer and video player with keyboard navigation
- **Metadata Extraction** — Automatically parses ComfyUI workflow metadata embedded in PNG files (prompt, workflow JSON)
- **Search & Filter** — Search by prompt text, filter by checkpoint model, sampler, and more
- **Click-to-Copy** — Copy any metadata value (prompts, seeds, model names) with a single click

## Tech Stack

- **Backend**: Python / FastAPI / SQLAlchemy / PostgreSQL
- **Frontend**: React / TypeScript / Vite / TailwindCSS
- **Infrastructure**: Docker Compose

## Quick Start

### Prerequisites

- [Docker](https://docs.docker.com/get-docker/) and [Docker Compose](https://docs.docker.com/compose/install/)

### Setup

1. **Clone the repository**
   ```bash
   git clone https://github.com/fred-aghq/comfy-gallery.git
   cd comfy-gallery
   ```

2. **Configure your media path**
   ```bash
   cp .env.example .env
   ```
   Edit `.env` and set `MEDIA_ROOT` to your ComfyUI output directory:
   ```env
   MEDIA_ROOT=/home/youruser/ComfyUI/output
   ```

3. **Start the stack**
   ```bash
   docker compose up -d
   ```

4. **Open the gallery**
   
   Visit [http://localhost:5173](http://localhost:5173)

The gallery will automatically scan your media directory on startup and extract metadata from all supported files.

### Rescan

To pick up new files, click the **Rescan** button in the header, or call the API:
```bash
curl -X POST http://localhost:8000/api/scan
```

## Development

The dev environment uses hot-reload for both backend and frontend:

```bash
docker compose up
```

- **Frontend**: [http://localhost:5173](http://localhost:5173) (Vite dev server with HMR)
- **Backend API**: [http://localhost:8000](http://localhost:8000) (FastAPI with auto-reload)
- **API docs**: [http://localhost:8000/docs](http://localhost:8000/docs) (Swagger UI)

### Running Backend Tests

The backend has a test suite covering metadata extraction, file scanning, thumbnails, and API routes. Tests use in-memory SQLite so they run without Docker or PostgreSQL:

```bash
cd backend
pip install -e ".[dev]"
pytest tests/ -v
```

## Production

```bash
cp .env.example .env
# Edit .env with production values
docker compose -f docker-compose.prod.yml up -d --build
```

The gallery will be available on port 80 (or whatever `PORT` is set to in `.env`).

## Supported Formats

| Type | Extensions |
|------|-----------|
| Images | `.png`, `.jpg`, `.jpeg`, `.webp` |
| Videos | `.mp4`, `.webm`, `.mov`, `.avi`, `.mkv` |

## Metadata

ComfyUI embeds workflow metadata in PNG files as JSON in the image's tEXt chunks. The gallery extracts and indexes:

- **Checkpoint / Model name**
- **Positive & negative prompts**
- **Sampler, scheduler, CFG scale, steps, seed**
- **LoRA names**
- **Image/video dimensions**
- **Full raw workflow JSON** (viewable in the media viewer)

For non-PNG formats, the gallery also checks for sidecar `.json` files and EXIF metadata.

## API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/api/health` | Health check |
| `GET` | `/api/media` | List media (paginated, filterable) |
| `GET` | `/api/media/{id}` | Get single media item |
| `GET` | `/api/media/folders/tree` | Get folder tree structure |
| `GET` | `/api/media/file/{path}` | Serve original media file |
| `GET` | `/api/thumbnails/{name}` | Serve thumbnail |
| `POST` | `/api/scan` | Trigger directory rescan |
